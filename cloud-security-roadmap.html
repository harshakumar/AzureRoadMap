Section 01

Azure vs AWS — Start with Azure

For the UK job market in 2026, Azure is the clear choice. It dominates UK enterprise, public sector, and financial services. The Azure cert path has the most direct progression into cloud security of any vendor.

The verdict: Start with Azure. Add AWS Security Specialty after landing your first role (year 2). Multi-cloud adds 15–25% salary premium at senior level — build it when you have real-world context to apply it.

Azure — why it wins for the UK
✓Dominates UK public sector — NHS, HMRC, MOD, local councils all run Azure. Government cleared roles pay a significant premium.

→Start with AWS Cloud Practitioner (CLF-C02) after AZ-500 if you want breadth, then AWS Security Specialty year 2.
→Adrian Cantrill's AWS courses are the gold standard for paid content. Bookmark now, use in year 2.

→Being multi-cloud literate adds 15–25% salary premium at senior level. Build it after you have the foundation.

Section 06

Daily Study Schedule

This schedule is for someone currently employed. It totals 25 hrs/week — aggressive but essential. The morning block is non-negotiable. Protect it like a critical client meeting.

Weekday schedule — Monday to Friday
6:00–7:00 AM
Theory block. Video course or Microsoft Learn. Phone on silent. Before the day starts. This single hour is where most of your learning happens.
7:00–7:30 AM
Consolidation. Write 3–5 bullet points in your own words summarising what you just learned. If you cannot explain it simply, you have not understood it.
7:00–9:00 PM
Labs + practice questions. Alternate: Mon/Wed/Fri = Azure portal hands-on labs. Tue/Thu = Whizlabs mock questions and exam prep review.
9:00–9:30 PM
Anki flashcard review. 15 minutes of spaced repetition. Create cards throughout the day for anything you struggle with. Free Anki app.
Weekend schedule — Saturday & Sunday
Sat 9 AM–12
Deep lab session. 3 continuous hours in Azure. Build something real. Document it as a GitHub portfolio piece while you work.
Sat 2–4 PM
Full timed mock exam. No pausing, no notes. Simulate real exam conditions. Review every wrong answer against official Microsoft documentation.
Sun 9–11 AM
Weak area focus. Review the lowest-scoring domain from Saturday's mock. 2 hours of targeted, deep study on problem areas only.
Sun afternoon
Complete rest. No studying. This is part of the plan — cognitive consolidation happens during rest. Non-negotiable. Do not feel guilty.
Weekly hours by phase
Month 120 hrs/week. AZ-900 and SC-900 are lighter exams. Use the manageable pace to build solid study habits and set up your Azure lab environment properly.
Month 225 hrs/week. AZ-500 is the hardest exam of this plan. Protect every study session. This month determines whether you pass in August.
Month 325 hrs/week. Complete Domains 3 & 4 in weeks 9–10, then 100% mock exams weeks 11–12. Sit AZ-500 on schedule — do not postpone.
Month 420 hrs/week studying + 1hr/day job applications. From week 3 of month 4: 60% study, 40% applying and interview prep.
Month 510 hrs/week studying + full-time job hunt. Study only to fill gaps exposed by interviews. Portfolio and interview prep take full priority.
The one rule that matters most: Never miss the 6 AM morning block two days in a row. Miss it once — fine, catch up in the evening. Miss it twice — your pace collapses and the exam dates slip. Pick a fixed time and treat it as a non-negotiable appointment with your future career.
Section 07

Portfolio Projects — Your Backend Dev Advantage

No pure security candidate coming from a non-engineering background can build a secure CI/CD pipeline from scratch. These four projects demonstrate skills that most security applicants simply cannot produce.

Project 01 — Month 2
Azure Security Baseline (Terraform IaC)
Deploy a fully hardened Azure subscription using Terraform. Include: NSGs with least-privilege rules, Azure Key Vault with access policies, Defender for Cloud enabled and configured, Diagnostic Settings forwarding all logs to Log Analytics, Microsoft Sentinel connected. Run it, capture the Defender for Cloud Secure Score, document every security decision made and why. This demonstrates both cloud security knowledge AND engineering capability — a combination most security candidates lack.
Project 02 — Month 3
Microsoft Sentinel Detection Lab
Working Sentinel workspace with 5 custom KQL analytics rules targeting real-world attack patterns (brute force, impossible travel, privilege escalation, data exfiltration, lateral movement). Build 2 Sentinel workbooks visualising the security posture. Simulate each attack, capture the generated incident, document your investigation following the PICERL incident response framework. Publish all KQL queries on GitHub with line-by-line explanations of the detection logic.
Project 03 — Month 4
Zero Trust Architecture Implementation Guide
A 5-page written architecture document showing how you would implement zero trust for a mid-size organisation on Azure. Cover: Conditional Access policy set, PIM configuration for privileged roles, network micro-segmentation design, Defender for Endpoint deployment, Sentinel monitoring strategy, and compliance reporting. Include architecture diagrams created in draw.io or Excalidraw. This demonstrates strategic security thinking, not just technical configuration skills.
Project 04 — Month 5 · The differentiator
Secure CI/CD Pipeline (your killer advantage)
GitHub Actions pipeline incorporating: Checkov and tfsec for IaC security scanning, CodeQL for static application security testing, Trivy for container image vulnerability scanning, GitLeaks for secrets detection, and automated SBOM (Software Bill of Materials) generation. Add a policy gate that fails the pipeline if critical vulnerabilities are found. This is the project no pure security candidate can produce. It demonstrates exactly why your backend engineering background makes you more effective as a security engineer.
