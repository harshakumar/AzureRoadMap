Cloud Concepts 25–30% of exam

1.1 What Is Cloud Computing?

Cloud computing delivers computing services over the internet — servers, storage, databases, networking, software — enabling faster innovation and flexible, pay-as-you-go resources.

💡
NIST 5 characteristics: On-demand self-service · Broad network access · Resource pooling (multi-tenancy) · Rapid elasticity · Measured service (metered billing)

1.2 Cloud Service Models

ModelYou ManageProvider ManagesExamples
IaaS — Infrastructure as a ServiceOS, middleware, runtime, apps, dataHardware, network, virtualizationAzure VMs, Azure Disks
PaaS — Platform as a ServiceApps, dataEverything underneathAzure App Service, Azure SQL
SaaS — Software as a ServiceNothing (just use it)EverythingMicrosoft 365, Dynamics 365
⚠️
Shared Responsibility Model: As you move IaaS → PaaS → SaaS, more shifts to Microsoft. Security of the cloud = Microsoft's. Security in the cloud = yours (varies by model).

1.3 Deployment Models

🌐
Public Cloud
Resources owned by a third-party, shared over the internet. Best for startups and variable workloads.
🏢
Private Cloud
Exclusive to one organization; on-prem or hosted. Best for regulated industries and sensitive data.
🔀
Hybrid Cloud
Combines public + private, connected by tech. Used for cloud bursting and phased migrations.
☁️
Multi-Cloud
Multiple cloud providers. Avoids vendor lock-in; enables best-of-breed service selection.

Azure Architecture & Infrastructure

2.1 Global Infrastructure Hierarchy

Azure Global Infrastructure
├── Geographies  (e.g. United States, Europe, Asia Pacific)
│   └── Regions  (e.g. East US, West Europe, Southeast Asia)
│       ├── Region Pairs  (paired for disaster recovery, ≥300 mi apart)
│       └── Availability Zones
│           └── Zone 1 / Zone 2 / Zone 3  (physically separate datacenters)
│               └── Datacenters
ConceptDescription
GeographyDiscrete market with 2+ regions; preserves data residency and compliance
RegionSet of datacenters within a latency-defined perimeter; every resource lives in a region
Region PairEach region paired with another ≥300 mi away in same geography for replication/recovery
Availability ZonePhysically separate datacenters within a region — independent power, cooling, networking
Sovereign RegionsIsolated instances for compliance: Azure Government (US), Azure China (21Vianet)

2.2 Availability Zones vs Availability Sets

Availability ZonesAvailability Sets
ScopeDifferent datacenters within a regionDifferent racks within a datacenter
Protects againstEntire datacenter failureRack-level (power/network) failure
SLA99.99%99.95%
ConceptsZonal, Zone-redundant, Non-regionalFault Domains (FD) + Update Domains (UD)
✅
Availability Sets: Fault Domains = separate physical racks (max 2–3). Update Domains = groups restarted together during maintenance (default 5, max 20).

2.3 Resource Organization Hierarchy

Azure AD Tenant  (identity boundary)
└── Management Groups  (policy at scale; up to 6 levels)
    └── Subscriptions  (billing + access control boundary)
        └── Resource Groups  (logical container; must belong to 1 subscription)
            └── Resources  (VMs, storage, databases, etc.)
LevelKey Facts
Management GroupsContainer for subscriptions; apply Policy & RBAC at scale; root group at top
SubscriptionsBilling boundary AND access control boundary. Common splits: per environment / per dept
Resource GroupsEvery resource in exactly one RG; can span regions; delete RG = delete all resources inside
ResourcesActual Azure services: VMs, storage accounts, SQL DBs, etc.

2.4 Azure Resource Manager (ARM)

ARM is the deployment and management layer for all Azure services. Every tool (portal, CLI, PowerShell, REST) goes through ARM.

Consistent management layer Declarative templates (ARM/Bicep) RBAC built-in Tagging support Dependency management

Compute Services

3.1 Azure Virtual Machines (IaaS)

Full control over OS and configuration. You are responsible for OS patching, software, and security.

Availability OptionProtects AgainstSLA
Single VM + Premium SSDHardware failure99.9%
Availability Set (FDs + UDs)Rack-level failure99.95%

💡

Azure Dedicated Hosts: Physical server dedicated to your org — useful for compliance/licensing requirements.

3.2 Azure App Service (PaaS)

Fully managed platform for web apps, REST APIs, and mobile backends. No server management required.

.NET / Java / Node.js / Python / PHP / Ruby Built-in autoscale CI/CD integration Deployment slots Custom domains + SSL
✅
Deployment Slots: Separate environments (staging/prod) that can be swapped for zero-downtime deployments.

Networking Services

4.1 Azure Virtual Network (VNet)

Foundation of Azure networking — a private network scoped to one region and subscription. Resources in the same VNet communicate by default.

Isolation & segmentation via subnets Internet communication Service/Private Endpoints On-prem connectivity NSGs & route tables

4.2 Connectivity Options

ServiceTravels OverSpeedUse When
VPN GatewayPublic internet (encrypted)Up to ~10 GbpsCost-effective, moderate bandwidth, remote access
ExpressRoutePrivate dedicated circuit50 Mbps – 100 GbpsHigh bandwidth, consistent latency, compliance
VNet PeeringMicrosoft backbone (private)Full bandwidthConnect two VNets in same or different regions
⚠️
VNet Peering is non-transitive by default. A↔B and B↔C does NOT mean A↔C. You need explicit peering A↔C or a hub/spoke with Azure Firewall.

4.3 Load Balancing Comparison

ServiceLayerScopeKey Feature
Load BalancerLayer 4 (TCP/UDP)RegionalBasic load balancing; public or internal
Application GatewayLayer 7 (HTTP/S)RegionalURL-based routing, SSL termination, WAF
Traffic ManagerDNS (Layer 7)GlobalDNS-based routing; Performance/Priority/Geo methods
Azure Front DoorLayer 7Global edgeCDN + WAF + Anycast routing; fastest failover

4.4 Network Security Groups (NSGs)

Filter inbound/outbound traffic at the subnet or NIC level. Rules have priority (lower number = processed first), source/dest, port, protocol, and allow/deny action.

💡
Default rules: allow inbound from VNet + Azure Load Balancer, deny all other inbound internet. You must explicitly open ports you need.

Storage Services

5.1 Azure Storage Account Services

📦
Blob Storage
Object storage for unstructured data: images, videos, logs, backups. Block, Append, and Page blob types.
📁
Azure Files
Managed cloud file shares via SMB / NFS. Mount on Windows, Linux, macOS simultaneously.
📬
Queue Storage
Async message queue. Messages up to 64 KB. Decouples application components (producer/consumer).
🗃️
Table Storage
NoSQL key-attribute schemaless store. Low cost for large structured datasets.

OptionCopiesProtects Against

Identity, Access & Security

6.1 Microsoft Entra ID (Azure AD)

Microsoft's cloud-based identity and access management service. Not the same as on-premises Windows Active Directory.

Traditional AD (on-prem)Microsoft Entra ID (cloud)
ProtocolLDAP, Kerberos, NTLMSAML, OAuth 2.0, OpenID Connect
StructureDomains, OUs, GPOsFlat — no OUs or GPOs
LocationYour datacentersMicrosoft's cloud
Primary UseCorp network resourcesCloud/web applications

6.2 Authentication vs Authorization

🔑
Authentication (AuthN)
Verifying who you are. Methods: password, MFA, certificate, biometrics.
🛡️
Authorization (AuthZ)
Verifying what you can do. Controlled via RBAC, policies, and permissions.

6.3 Multi-Factor Authentication (MFA)

Requires two or more of: something you know (password/PIN), something you have (phone/authenticator app), something you are (fingerprint/face).

6.4 Conditional Access

Enforce policies after first-factor authentication. Uses an If–Then model: IF user comes from untrusted location, THEN require MFA.

⚠️
Conditional Access requires Microsoft Entra ID P1 or higher (included in Microsoft 365 E3/E5).

6.5 Role-Based Access Control (RBAC)

Built-in RolePermissions
OwnerFull access + manage access assignments
ContributorFull access, cannot manage access
ReaderView only — no modifications
User Access AdminManage access only — no resource changes
💡
RBAC is additive — multiple role assignments combine. Deny assignments (from Policy) override RBAC allows. Apply at management group, subscription, RG, or resource scope — inherited downward.

6.6 Zero Trust Model

🔍
Verify Explicitly
Always authenticate and authorise based on all available data points — identity, location, device, service, workload.
🎯
Least Privilege Access
Limit access with Just-In-Time (JIT) and Just-Enough-Access (JEA). Minimise blast radius.
💥
Assume Breach
Segment access, verify end-to-end encryption, use analytics to detect and respond to anomalies.

6.7 Defence in Depth

Layer (outer → inner)Controls
Physical SecurityDatacentre access controls, guards, CCTV
Identity & AccessEntra ID, MFA, RBAC, Conditional Access
PerimeterDDoS Protection, Azure Firewall
NetworkNSGs, VNet segmentation, VPN
ComputeOS patches, endpoint protection, AV

Cost Management & Governance 30–35% of exam

7.1 Factors Affecting Azure Cost

Resource type & tier Amount consumed (hours, GB, operations) Region (prices vary) Outbound data transfer Subscription type (PAYG vs EA vs CSP)
💡
Inbound data is free. Outbound (egress) data incurs charges after a free allowance per region.

7.2 Cost Reduction Strategies

StrategySavingsDetails
Azure ReservationsUp to 72%Commit to 1 or 3 years upfront; available for VMs, SQL, Cosmos DB, storage
Azure Hybrid BenefitUp to 40%Use existing Windows Server / SQL Server licenses with Software Assurance
Spot VMsUp to 90%Unused Azure capacity; VMs evicted with 30s notice — use for interruptible workloads
Dev/Test PricingVariesDiscounted rates for non-production (requires Visual Studio subscription)
Right-SizingVariesMatch VM size to actual workload; Azure Advisor provides recommendations

7.3 Cost Tools

ToolURLPurpose
Pricing Calculatorazure.microsoft.com/pricing/calculatorEstimate cost of Azure services before deploying
TCO Calculatorazure.microsoft.com/pricing/tcoCompare on-premises vs Azure cost over 3–5 years
Cost Management + BillingAzure portalAnalyse spending, set budgets, alerts, download invoices

7.4 Governance Tools

ToolWhat It Does
Azure TagsName-value pairs on resources for cost allocation, automation, ownership. Not inherited by default.
Azure PolicyEnforce standards at scale. Effects: Deny, Audit, Append, Modify, DeployIfNotExists. Compliance dashboard included.
Resource LocksCanNotDelete (read + modify, no delete) or ReadOnly (read only). Inherited by child resources.
Azure BlueprintsPackage policies + RBAC + ARM templates for repeatable governed deployments to subscriptions.
Microsoft PurviewUnified data governance: data map, catalog, lineage, information protection, compliance manager.
⚠️
Azure Policy vs RBAC: Policy controls what resources look like (properties, configurations). RBAC controls what users can do (actions on resources). They work together, not instead of each other.

7.5 Service Trust Portal

Visit servicetrust.microsoft.com for compliance content: audit reports, certifications, whitepapers for ISO 27001, SOC, HIPAA, GDPR, etc.

💰
